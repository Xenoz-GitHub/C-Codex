C++ Codex

Simple C++ reference. Syntax + explanation + example.
For beginners. For pros. Everyone.

01 Basics — what you write first

Comments //

Comments are notes for humans. The compiler ignores them. Use // for one line. Use /* */ for multiple lines.

// This is a comment
int x = 5;  // code can have comments too

/* Multi-line
   comment block */

Variables & Types int

A variable stores a value. You must say what type: int (whole number), double (decimal), char (letter), bool (true/false). auto lets the compiler guess the type.

int age = 25;               // whole number
double price = 9.99;         // decimal number
char grade = 'A';            // single letter
bool ready = true;            // true or false
auto x = 42;                 // compiler figures out int
const int MAX = 100;          // cannot change this value

Input / Output cin/cout

cout prints to the screen. cin reads what the user types. cerr prints errors. getline reads a whole line including spaces.

cout << "Hello";               // print to screen
cout << "Value: " << x << "\n";  // print multiple things
cin >> name;                     // read one word
getline(cin, line);               // read whole line
cerr << "Oops, error!";          // print error message

Operators + - * /

Operators do things to values. Math: + - * / %. Compare: == != < >. Logic: && || !. Ternary ?: is a short if/else.

int sum = a + b;           // add
bool eq = (a == b);        // equal to?
bool ok = a && b;          // AND: both true?
int mx = (a > b) ? a : b;  // if a>b use a, else b
a += 5;                     // same as a = a + 5

Enum & Struct enum/struct

enum gives names to numbers. enum class prevents name clashes. struct groups related data together.

enum Color { Red, Green, Blue };  // Red=0, Green=1, Blue=2
Color c = Red;

enum class Status { Ok, Error };   // scoped: must use Status::Ok
Status s = Status::Ok;

struct Point {                     // groups x and y together
    double x, y;
};
Point p = {3.0, 4.0};

Type Casting cast

Change one type to another. static_cast is the safe one you'll use most. dynamic_cast works with class hierarchies. Avoid C-style (type)val.

double d = static_cast<double>(a) / b;  // int → double

Derived* d = dynamic_cast<Derived*>(base);
if (d) { /* cast worked */ }             // null = failed

const_cast<char*>(str);               // remove const
reinterpret_cast<int*>(addr);          // re-interpret bits

Namespaces namespace

Namespaces group code so names don't clash. std::cout means "cout from the std namespace". using namespace brings all names into view — use it carefully.

namespace MyCode {
    int doSomething() { return 42; }
}
MyCode::doSomething();                // call with namespace

using namespace MyCode;               // now just doSomething()
namespace M = MyCode;                 // shorter alias

Type Aliases using

Give a type a new name. using (C++11) is better than typedef — it reads left-to-right and works with templates.

using ulong = unsigned long;          // alias for long name
ulong big = 1000000;

using StringVec = std::vector<std::string>;  // alias for complex type
StringVec names;

Raw Strings R"()"

Raw strings ignore escape characters. Perfect for file paths, regex, or multi-line text. Write R"(...)" and everything inside is literal.

std::string path = R"(C:\Program Files\MyApp)";
// without raw: "C:\\Program Files\\MyApp" — ugh

std::string html = R"(<tag attr="value">text</tag>)";
// no need to escape quotes inside

02 Control Flow — make decisions, repeat tasks

If / Else if

Run code only when a condition is true. else if checks another condition. else runs when nothing else matched.

if (score >= 90) {
    grade = 'A';
} else if (score >= 80) {
    grade = 'B';
} else {
    grade = 'F';
}

// C++17: declare variable inside if
if (int result = compute(); result > 0) {
    // result is valid here
}

Switch switch

Pick one option from many values. Works with integers and enums. Use break to stop — without it, execution falls through to the next case.

switch (day) {
    case 1: cout << "Monday";    break;
    case 2: cout << "Tuesday";   break;
    case 3:
    case 4: cout << "Midweek";    break;  // 3 and 4 same
    default: cout << "Unknown";
}

For & While for

Repeat code. for when you know the count. while when you don't. do-while runs at least once.

for (int i = 0; i < 5; i++) {        // run 5 times
    cout << i;                        // prints 01234
}

while (x > 0) {                      // runs until x <= 0
    x--;
}

do {                                  // always runs at least once
    cout << x;
} while (x > 10);

Range For : range

Loop over every item in a collection. Use auto& to change items. Use const auto& to just read (no copying).

std::vector<int> numbers = {10, 20, 30};

for (int n : numbers) {              // copies each value
    cout << n;
}
for (auto& n : numbers) {           // modify original
    n *= 2;
}
for (const auto& n : numbers) {     // read only, no copy
    cout << n;
}

Recursion recurse

A function that calls itself. Always needs a base case to stop, or it runs forever (stack overflow). Good for problems that break into similar sub-problems.

int factorial(int n) {
    if (n <= 1) return 1;           // base case: stop here
    return n * factorial(n - 1);   // n! = n × (n-1)!
}
// factorial(5) → 5×4×3×2×1 = 120

int fib(int n) {
    if (n <= 1) return n;            // fib(0)=0, fib(1)=1
    return fib(n-1) + fib(n-2);
}

03 Data & Memory — store, point, allocate

Arrays & Strings []

int arr[5] is a fixed-size list. Use std::vector for a dynamic list. std::string holds text and has many helper functions.

int arr[3] = {10, 20, 30};       // fixed array of 3
arr[0] = 99;                       // change first element

std::vector<int> v = {1,2,3};     // dynamic array
v.push_back(4);                    // add item at end

std::string name = "Alice";
name.size();                        // how many characters?
name += " Smith";                  // add to string

Pointers & References * &

A pointer (*) stores a memory address. A reference (&) is an alias — like a different name for the same variable. References are safer.

int x = 42;
int* p = &x;               // p points to x
cout << *p;                // read what p points to: 42

int& ref = x;              // ref is another name for x
ref = 99;                   // changes x to 99!

p = nullptr;                // pointer to nothing
if (p) { }                  // false if p is nullptr

Dynamic Memory new/delete

new creates something on the heap (lives until you delete it). Always match new with delete. Better yet, use smart pointers — they auto-delete.

int* p = new int(42);         // allocate on heap
cout << *p;                     // use it
delete p;                        // free it!

int* arr = new int[100];
delete[] arr;                   // array delete

// Smart pointers — auto cleanup
auto u = std::make_unique<int>(5);
auto s = std::make_shared<int>(10);

04 Functions — reusable code blocks

Functions returnType name()

A function is a named block of code that runs when you call it. Specify what it returns (or void for nothing). Arguments are inputs.

int add(int a, int b) {          // returns int
    return a + b;
}
void greet(std::string name) {   // returns nothing
    cout << "Hi " << name;
}
int result = add(3, 4);           // call it

// Default arguments
void log(std::string msg, int level = 0);  // level defaults to 0

Lambda [](){}

A lambda is a short function you write inline. [ ] captures outside variables. [=] captures by value. [&] captures by reference.

auto square = [](int x) { return x * x; };
cout << square(5);                   // 25

int offset = 10;
auto add = [offset](int x) {          // captures offset by value
    return x + offset;
};

auto counter = [n = 0]() mutable {     // init capture
    return n++;                        // mutable lets n change
};

Function Pointers fn*

Store a function in a variable. The type must match the function signature. std::function is more flexible — works with lambdas too.

int (*op)(int, int) = &add;  // pointer to add function
cout << op(3, 4);               // call via pointer: 7

// std::function — any callable
std::function<int(int)> f = square;
cout << f(3);                    // works with lambdas too

Inline & constexpr inline

inline suggests the compiler copy the code directly (no function call). constexpr tells the compiler to calculate at compile-time when possible.

inline int square(int x) {        // may avoid call overhead
    return x * x;
}

constexpr int fact(int n) {      // compute at compile-time
    return (n <= 1) ? 1 : n * fact(n - 1);
}
constexpr int VAL = fact(5);

=default & =delete default/delete

= default says "use the compiler's version" (for constructors, destructors). = delete says "ban this" — useful to prevent copying.

class NoCopy {
public:
    NoCopy() = default;                 // use default constructor
    ~NoCopy() = default;
    NoCopy(const NoCopy&) = delete;       // forbid copying
    NoCopy& operator=(const NoCopy&) = delete;
};

void onlyInt(int) { }                 // OK
void onlyInt(double) = delete;        // calling with double = error

decltype & typeid decltype

decltype figures out the type of an expression at compile-time. typeid (from <typeinfo>) checks types at runtime.

int x = 42;
decltype(x) y = 10;                 // y is int (same as x)

auto add(auto a, auto b) -> decltype(a + b) {
    return a + b;                     // trailing return type
}

if (typeid(x) == typeid(int)) { }   // runtime type check

05 OOP — classes, inheritance, polymorphism

Classes class

A class bundles data and functions together. private means only the class can touch it. public means anyone can. Constructors run when you create an object.

class Player {
    int hp;                              // private by default
public:
    Player(int h) : hp(h) { }           // constructor sets hp
    void takeDamage(int d) { hp -= d; }
    int getHP() { return hp; }
};  // don't forget the semicolon!

Player p(100);                         // create Player with 100 HP
p.takeDamage(20);                    // HP is now 80

Inheritance :

One class can inherit from another. The child gets everything the parent has. Use public to keep the parent's access levels. A virtual destructor ensures proper cleanup.

class Entity {                      // base class
public:
    int id;
    virtual ~Entity() = default;      // virtual destructor
};

class Player : public Entity { };    // Player IS-AN Entity
Player p;
p.id = 1;                            // inherited from Entity

Polymorphism virtual

If a function is virtual, the child can replace it. A pure virtual function (= 0) forces children to provide it — the class becomes abstract (can't create it directly).

class Shape {
public:
    virtual double area() const = 0;   // pure virtual
};

class Circle : public Shape {
    double area() override {          // must provide this
        return 3.14 * r * r;
    }
};

Shape* s = new Circle(5);
cout << s->area();                   // calls Circle's version

Copy Control rule of 5

If your class uses raw memory (new), define these 5: destructor, copy constructor, copy assignment, move constructor, move assignment. Otherwise you'll get bugs.

class Buffer {
    int* data;  int size;
public:
    Buffer(int s) : size(s), data(new int[s]) { }
    ~Buffer() { delete[] data; }                // 1
    Buffer(const Buffer& o)                     // 2: copy ctor
        : size(o.size), data(new int[o.size]) {
        copy(o.data, o.data+size, data);
    }
    Buffer& operator=(const Buffer&) { }     // 3: copy assign
    Buffer(Buffer&&) noexcept { }              // 4: move ctor
    Buffer& operator=(Buffer&&) noexcept { }  // 5: move assign
};

Operator Overloading operator

Give operators like +, ==, << a special meaning for your class. Make your types feel like built-in ones.

struct Vec2 {
    float x, y;
    Vec2 operator+(const Vec2& o) const {
        return {x + o.x, y + o.y};
    }
    bool operator==(const Vec2& o) const {
        return x == o.x && y == o.y;
    }
};
Vec2 a{1,2}, b{3,4};
auto c = a + b;                     // {4, 6}

Static static

static on a class member means all objects share the same copy (it belongs to the class, not any one object). static in a function keeps the value between calls.

class App {
public:
    static int count;                  // shared by all App objects
};
int App::count = 0;                    // define it (once!)

App a, b;
a.count = 5;
cout << b.count;                      // 5 — same value!

int nextID() {
    static int id = 0;               // keeps its value
    return id++;                       // 0, 1, 2, 3...
}

Friend friend

Sometimes an outside function needs to see private data. Declare it a friend and it gets access. Use sparingly — it breaks encapsulation.

class Account {
    int balance;
public:
    Account(int b) : balance(b) { }
    friend bool isOverdrawn(const Account& a);  // friend
};

bool isOverdrawn(const Account& a) {
    return a.balance < 0;              // can see private balance!
}

explicit explicit

explicit stops the compiler from using your constructor to secretly convert types. Without it, String s = 5; might compile and that's confusing.

class String {
public:
    explicit String(int n) { }         // no implicit int→String
};
String s = 5;                           // ERROR: explicit blocks this
String s(5);                           // OK: direct construction

Conversion Operators operator T

Tell C++ how to convert your class to another type. Use explicit (C++11) to prevent accidental conversions.

struct Rational {
    int num, den;
    operator double() const {          // convert to double
        return (double)num / den;
    }
};
Rational r{3, 4};
double d = r;                          // automatically: 0.75

06 Modern C++ — C++11/14/17/20 features

Templates <T>

Write code that works with any type. The compiler generates the real version when you use it. Like a stencil — stamp out the same code for int, double, string, etc.

template<typename T>
T max(T a, T b) { return (a > b) ? a : b; }

max(3, 7);       // T = int
max(3.14, 2.5);  // T = double
max("cat", "dog"); // T = const char*

Exceptions try/catch

When something goes wrong, throw an exception. It travels up the call stack until a catch handles it. If nothing catches it, the program crashes.

try {
    if (file_not_found)
        throw std::runtime_error("file missing");
} catch (const std::exception& e) {
    cout << "Error: " << e.what();  // describe the problem
} catch (...) {
    cout << "Unknown error";          // catch anything
}

STL Containers vector/map

The standard library gives you ready-to-use data structures. vector = dynamic array. map = key→value lookup. set = sorted unique items.

std::vector<int> v = {1,2,3};
v.push_back(4);                     // add to end
v.pop_back();                       // remove last

std::map<std::string, int> ages;
ages["Alice"] = 30;                   // store value by key
cout << ages["Alice"];               // look up

std::set<int> s = {3,1,2,1};         // {1, 2, 3} (no duplicates)

Move Semantics &&

Instead of copying big data, you can move it — like transferring ownership. std::move says "I won't need this anymore, take its resources." Much faster than copying.

std::vector<int> a = {1,2,3};
std::vector<int> b = std::move(a);   // move, not copy!
// a is now empty, b owns the data

// Move constructor — take ownership from temporary
class MyType {
    int* ptr;
public:
    MyType(MyType&& other) noexcept
        : ptr(other.ptr) {
        other.ptr = nullptr;           // disarm the source
    }
};

Iterators & Algorithms sort/find

Iterators let you walk through containers without caring about the type. Algorithms (sort, find, for_each) work on any container via iterators.

std::vector<int> v = {4,1,3,2};
std::sort(v.begin(), v.end());        // sort: 1 2 3 4

auto it = std::find(v.begin(),
                    v.end(), 3);       // find value 3
if (it != v.end()) { /* found! */ }

std::for_each(v.begin(), v.end(),
    [](int x) { cout << x; });        // do for each

optional & variant C++17

std::optional either has a value or doesn't — no more magic numbers for "no result". std::variant holds one type from a list (like a type-safe union).

std::optional<int> parse(std::string s) {
    if (s.empty()) return {};            // no value
    return stoi(s);                     // has value
}
if (auto v = parse("42")) cout << *v;

std::variant<int, float, std::string> v;
v = "hello";                            // holds string
cout << std::get<std::string>(v);    // extracts "hello"

Structured Bindings C++17

Unpack tuples, pairs, structs into named variables in one line. Cleaner than std::get or accessing .first/.second.

// With a map
for (const auto& [key, value] : my_map) {
    cout << key << ": " << value;
}

// With a tuple
auto [x, y, z] = tuple{1, 2.5, "hi"};
// x=1, y=2.5, z="hi"

// With a struct
struct Point { int x, y; };
Point p{3, 4};
auto [a, b] = p;                    // a=3, b=4

pair & tuple pair/tuple

std::pair holds two values of any types. std::tuple (C++11) holds any number. Perfect for returning multiple values from a function.

std::pair<int, std::string> p = {1, "one"};
cout << p.first << p.second;         // .first .second

std::tuple<int, double> getValues() {
    return {42, 3.14};
}
auto [n, d] = getValues();             // unpack with structured binding

Time (chrono) chrono

Measure time, sleep, work with durations. high_resolution_clock gives precise timing. C++20 adds calendar dates.

using namespace std::chrono;

// Measure time
auto start = high_resolution_clock::now();
// ... do work ...
auto end = high_resolution_clock::now();

auto ms = duration_cast<milliseconds>(end - start);
cout << ms.count() << "ms\n";

// Sleep for 1 second
std::this_thread::sleep_for(seconds(1));

string_view C++17

A string_view is like looking at a string without owning it. No copying! Fast for reading substrings, trimming, parsing.

std::string_view trim(std::string_view s) {
    while (s.front() == ' ') s.remove_prefix(1);
    while (s.back()  == ' ') s.remove_suffix(1);
    return s;                         // no memory allocation!
}
cout << trim("  hello  ");            // "hello"

// Accept any string type without copying
void log(std::string_view msg);

if constexpr C++17

Like a normal if, but the compiler checks it at compile-time. The "false" branch is discarded — it won't even be compiled. Great for templates.

template<typename T>
auto describe(const T& v) {
    if constexpr (std::is_integral_v<T>) {
        return "integer: " + std::to_string(v);
    } else {
        return std::string(v);          // for string-like types
    }
}
cout << describe(42);  // discarded string branch

Fold Expressions C++17

Apply an operator across all arguments in a parameter pack. No more writing recursive variadic templates just to sum things.

template<typename... Args>
auto sum(Args... args) {
    return (args + ...);               // add them all
}
// sum(1, 2, 3) expands to 1 + (2 + 3) = 6

template<typename... Args>
bool allTrue(Args... args) {
    return (... && args);              // AND all together
}
// allTrue(true, true, false) → false

07 System & Build — files, compiler, correctness

File I/O fstream

Read from and write to files. ifstream reads, ofstream writes, fstream does both. Always check if the file opened successfully.

std::ofstream out("data.txt");   // open for writing
if (out) {                           // did it open?
    out << "Hello file!\n";
    out.close();
}

std::ifstream in("data.txt");    // open for reading
std::string line;
while (std::getline(in, line))       // read line by line
    cout << line;

Preprocessor #

Lines starting with # are handled before compilation. #include pastes files. #define creates macros. Guards prevent including the same file twice.

#include <iostream>               // include standard header
#include "myheader.h"             // include your file

#define PI 3.14159                 // macro: replaces PI with value
// Use .vercel.appcheck
#pragma once                      // include guard (modern)

#ifdef _WIN32
    // Windows-specific code
#endif

Const const

const means "this won't change". The compiler enforces it. Read pointer declarations right-to-left: const int* = pointer to const int (can't change the int).

const int MAX = 100;               // MAX is always 100
// MAX = 200;  // error!

const int* p = &MAX;               // can't change *p
int const c = 5;                     // same as const int

// const method — promises not to modify the object
int getValue() const { return val; }

// const& param — read efficiently, no copy, no modification
void print(const std::string& s);

Union & Bit Fields union

A union stores different types in the same memory space (only one at a time). Bit fields let you specify exact bit counts for struct members.

union Value {
    int i;
    float f;
    char c[4];
};                                    // size = 4 bytes
Value v;
v.f = 3.14;                          // store as float

struct Flags {
    unsigned int a : 1;            // 1 bit
    unsigned int b : 3;            // 3 bits
    unsigned int c : 4;            // 4 bits
};                                     // packed into 1 byte

08 Advanced — Lvalues, concepts, placement

Lvalues & Rvalues & &&

An lvalue has an address (a variable). An rvalue is a temporary (42, a function result). && binds to rvalues — used for move semantics and perfect forwarding.

int x = 10;                    // x = lvalue, 10 = rvalue
int& lr = x;                    // lvalue reference
int&& rr = 20;                 // rvalue reference

// Forwarding reference — preserves lvalue/rvalue
template<typename T>
void wrapper(T&& v) {
    target(std::forward<T>(v));   // perfect forwarding
}

Concepts (C++20) requires

Say what types your template accepts, clearly. Concepts give better error messages than template soup. C++20 has built-in concepts like std::integral.

template<typename T>
concept Number = std::integral<T> || std::floating_point<T>;

template<Number T>                  // only numbers allowed
T add(T a, T b) { return a + b; }

// Abbreviated syntax
auto square(std::integral auto x) {  // short and clear
    return x * x;
}

Placement new placement

Build an object inside existing memory (no allocation). You must call the destructor manually. Used in custom allocators, game engines, embedded systems.

// Get raw memory
void* mem = operator new(sizeof(std::string));

// Construct object in that memory
std::string* s = new (mem) std::string("hello");

cout << *s;                          // use it

// Destroy manually
s->~basic_string();                   // call destructor
operator delete(mem);                // free raw memory
