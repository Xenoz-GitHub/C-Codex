C++ Codex

Every C++ syntax construct, distilled into concise examples.
Master the language — one snippet at a time.

01 Fundamentals — core syntax & building blocks

Comments //

Single-line (//) and multi-line (/* */) comments. Entirely ignored by the compiler — used to document intent.

// This is a single-line comment

/* This is a
   multi-line comment */

Variables & Data Types types

Fundamental types: int, double, char, bool, unsigned. Use auto for type deduction and const for immutables.

int age = 25;
double pi = 3.14159;
char grade = 'A';
bool ready = true;
auto result = 42;          // deduced as int
const int MAX = 100;       // read-only
unsigned int n = 255u;
long long big = 1e15;

Input / Output cin/cout

Console I/O via the <iostream> streams: cout (output), cin (input), cerr (error), clog (log).

cout << "Hello, " << name << "!\n";
cin >> name;
cerr << "Error occurred!" << endl;
clog << "Log: " << value << endl;

Operators + - * /

Arithmetic, relational, logical, bitwise, ternary (?:), and compound assignment operators.

int sum = a + b;         // arithmetic: + - * / %
bool eq = (a == b);      // relational: == != < > <= >=
bool ok = a && b || !c;  // logical: && || !
int bits = a & b | c;    // bitwise: & | ^ ~ << >>
int mx = (a > b) ? a : b; // ternary

a += 5;  a -= 2;            // compound assignment

Enums & Structs enum/struct

enum — named integer constants. enum class — scoped, type-safe (C++11). struct — grouped data members (default public).

enum Color { Red, Green, Blue };
enum class Status { Ok, Error };
Status s = Status::Ok;          // scoped access

struct Point {
    double x, y;
};
Point p = {3.0, 4.0};

Type Casting cast

Four C++ casts: static_cast (safe conversions), dynamic_cast (polymorphic downcast), const_cast (remove const), reinterpret_cast (reinterpret bits).

double d = static_cast<double>(a) / b;

Base* b = dynamic_cast<Base*>(ptr);
if (b) { }                 // null if cast fails

const_cast<char*>(str);      // remove const
reinterpret_cast<int*>(addr); // bit reinterpretation

Namespaces namespace

Group identifiers to avoid name collisions. Can be nested, aliased, or opened with using.

namespace Engine {
    namespace Math {
        int clamp(int v, int lo, int hi);
    }
}
using Engine::Math::clamp;

namespace EMath = Engine::Math;     // alias
using namespace std;               // brings all std names

Type Aliases typedef/using

typedef and using create synonyms for existing types. using is preferred (works with templates).

typedef unsigned long ulong;
using ulong = unsigned long;    // same, but clearer

template<typename T>
using Vec = std::vector<T>;    // alias template
Vec<int> v;

02 Control Flow — branching, iteration & recursion

Conditionals if/switch

if/else for boolean branching. switch/case for multi-way dispatch on integral/enum types.

if (x > 0) {
    // positive
} else if (x == 0) {
    // zero
} else {
    // negative
}

switch (grade) {                // integral/enum only
    case 'A': break;
    case 'B': break;
    default:  break;
}

Loops for/while

Four loop constructs: for, while, do-while, and range-based for (C++11) for iterating containers.

for (int i = 0; i < n; ++i) { }
while (condition) { }
do { } while (condition);

for (const auto& x : vec) { }   // range-based

Recursion recurse

A function calling itself. Every recursive call must move toward a base case to avoid infinite recursion (stack overflow).

int factorial(int n) {
    if (n <= 1) return 1;     // base case
    return n * factorial(n - 1); // recursive call
}

// Fibonacci
int fib(int n) {
    if (n <= 1) return n;
    return fib(n-1) + fib(n-2);
}

03 Data & Memory — storage, pointers, allocation

Arrays & Strings []

Fixed-size C-style arrays and the dynamic std::string class. Strings support concatenation, querying, and iteration.

int arr[5] = {1,2,3};       // rest → 0
arr[0] = 10;
int len = sizeof(arr) / sizeof(arr[0]);

char msg[] = "hello";          // C-string

std::string s = "C++";
s.size();  s.empty();
s += "17";                     // concatenation

Pointers & References * &

Pointers hold memory addresses. References are aliases — must be initialised and cannot be reassigned. Use nullptr for null pointers.

int x = 42;
int* p = &x;           // pointer: stores address
cout << *p;            // dereference → 42

int& ref = x;          // reference: alias for x
ref = 99;               // changes x to 99

p = nullptr;            // safe null pointer
if (p) { }             // false when p is null

Dynamic Memory new/delete

Heap allocation with new/delete. Prefer smart pointers (unique_ptr, shared_ptr) to avoid leaks.

int* p  = new int(42);    // single allocation
delete p;

int* ar = new int[10];    // array allocation
delete[] ar;

// Smart pointers (C++11, <memory>)
auto u = std::make_unique<int>(5);
auto s = std::make_shared<int>(10);

04 Functions — reusable logic, lambdas, callbacks

Functions ()

Declaration, definition, default arguments, overloading, void returns, and pass-by-reference vs pass-by-value.

int add(int a, int b) {
    return a + b;
}
void log(const std::string& msg, // pass by ref
           int level = 0) { }      // default param
double add(double a, double b);  // overload

Lambda Expressions []()

Inline anonymous functions (C++11). Capture variables by value [=], by reference [&], or selectively [x, &y].

auto square = [](int x) { return x * x; };
cout << square(5);               // 25

int offset = 10;
auto add_offset = [offset](int x) {
    return x + offset;
};

// Mutable lambda
auto counter = [n = 0]() mutable { return n++; };

Function Pointers fnptr

Store and pass functions as values. Use auto or explicit signature syntax. std::function generalises for lambdas.

int (*op)(int, int) = &add;  // function pointer
op(3, 4);                         // call via pointer

using Fn = int(*)(int, int);
Fn fn = &multiply;

// std::function (any callable)
std::function<int(int)> f = square;

Inline & constexpr inline/constexpr

inline hints the compiler to avoid function-call overhead. constexpr evaluates at compile-time where possible.

inline int square(int x) {
    return x * x;           // may inline at call site
}

constexpr int fact(int n) {
    return (n <= 1) ? 1 : n * fact(n - 1);
}
constexpr int VAL = fact(5);  // computed at compile-time

05 Object-Oriented Programming — classes, inheritance, polymorphism

Classes class

Encapsulation via access specifiers (private, public), constructors (with initialiser lists), destructors, and the this pointer.

class Player {
private:
    int hp;
public:
    Player(int h) : hp(h) { }     // init list
    ~Player() { }                  // destructor
    void takeDamage(int d) {
        hp -= d;
        this->hp = hp;            // explicit this
    }
};  // semicolon required

Inheritance :

Derive from base classes using :. Access modes: public (IS-A), protected, private. Use virtual for diamond-problem safety.

class Entity {
public:
    int id;
    virtual void update() { }
};

class Player : public Entity {
    // inherits id, can override update()
};

// Virtual inheritance prevents diamond duplicates
class D : public virtual B1,
          private B2 { };

Polymorphism virtual

Runtime dispatch via virtual functions. Pure virtual (= 0) makes a class abstract — cannot instantiate directly.

class Shape {
public:
    virtual double area() const = 0; // pure virtual
    virtual ~Shape() = default;
};
class Circle : public Shape {
    double area() override {
        return 3.14159 * r * r;
    }
};
Shape* s = new Circle(5);
s->area();                // dynamic dispatch

Copy Control rule of 5

If a class manages resources, define: destructor, copy ctor, copy assignment, move ctor, move assignment (Rule of Five).

class Buffer {
    int* data;
public:
    Buffer(size_t sz) : data(new int[sz]) { }
    ~Buffer() { delete[] data; }          // destructor
    Buffer(const Buffer& o)               // copy ctor
        : data(new int[size(o)]) { }
    Buffer& operator=(const Buffer&) { }  // copy assign
    Buffer(Buffer&&) noexcept { }        // move ctor
    Buffer& operator=(Buffer&&) noexcept { } // move assign
};

Operator Overloading operator

Define custom behaviour for operators (+, ==, <<, [], etc.) on user-defined types.

struct Vec2 {
    float x, y;
    Vec2 operator+(const Vec2& o) const {
        return {x + o.x, y + o.y};
    }
    bool operator==(const Vec2& o) const {
        return x == o.x && y == o.y;
    }
};
Vec2 a = {1,2}, b = {3,4};
auto c = a + b;               // calls operator+

Static Members static

static members belong to the class, not instances. static functions can only access static data. static local variables persist across calls.

class App {
public:
    static int count;               // declaration
    static void showCount() {
        cout << count;              // static fn
    }
};
int App::count = 0;                // definition

void counter() {
    static int n = 0;             // persists
    ++n;
}

Friend friend

A friend function or class can access private/protected members of the class that declares it as a friend.

class Account {
    int balance;
public:
    Account(int b) : balance(b) { }
    friend bool isOverdrawn(const Account& a);
    friend class Auditor;         // whole class
};

bool isOverdrawn(const Account& a) {
    return a.balance < 0;         // access private member
}

06 Modern C++ — generics, safety, algorithms, utilities

Templates <T>

Generic code with type parameters. Supports function templates, class templates, full/partial specialisation, and variadic templates (C++11).

template<typename T>
T max(T a, T b) { return (a > b) ? a : b; }

template<typename T, int N>
class Array { T data[N]; };

// Specialisation
template<> class Array<bool,8> { };

// Variadic (C++11)
template<typename... Args>
void print(Args... args) { }

Exceptions try/catch

try/catch/throw for error handling. Use catch(...) as a fallback. Standard exception classes derive from std::exception.

try {
    if (error)
        throw std::runtime_error("fail");
} catch (const std::exception& e) {
    cout << e.what();
} catch (...) {
    // catch any other exception
}

STL Containers vector/map/set

Workhorses of the Standard Library: vector (dynamic array), map (key-value), set (sorted unique), plus adaptors.

std::vector<int> v = {1,2,3};
v.push_back(4);                  // add to end
v.pop_back();                    // remove last

std::map<std::string, int> m;
m["key"] = 42;                    // insert/update

std::set<int> s = {3,1,2};       // sorted, unique
std::stack<int> stk;              // LIFO
std::queue<int> q;                 // FIFO

Move Semantics &&

Rvalue references (&&) enable move semantics — transferring resources without copying. std::move casts an lvalue to an rvalue.

std::vector<int> a = {1,2,3};
std::vector<int> b = std::move(a);  // a is now empty

class MyType {
public:
    MyType(MyType&& other) noexcept  // move ctor
        : ptr(other.ptr) {
        other.ptr = nullptr;         // pilfer resource
    }
};

Iterators & Algorithms sort/find

Iterators abstract container traversal. <algorithm> provides generic operations: sort, find, for_each, count_if, etc.

std::vector<int> v = {4,1,3,2};

std::sort(v.begin(), v.end());       // 1 2 3 4
auto it = std::find(v.begin(),
                    v.end(), 3);

std::for_each(v.begin(), v.end(),
    [](int x) { cout << x; });       // lambda

int cnt = std::count_if(v.begin(),
    v.end(), [](int x) { return x > 2; });

optional & variant C++17

std::optional — may or may not contain a value (safe nullable). std::variant — type-safe union holding one of several types.

std::optional<int> parseInt(const std::string& s) {
    if (s.empty()) return std::nullopt;
    return std::stoi(s);
}
auto val = parseInt("42");
if (val) cout << *val;            // 42

std::variant<int, float, std::string> v;
v = 10;                              // holds int
cout << std::get<int>(v);            // 10

Structured Bindings C++17

Decompose tuples, pairs, arrays, and structs into named variables in one declaration (C++17).

std::map<std::string, int> scores;
for (const auto& [name, score] : scores) {
    cout << name << ": " << score;
}

std::tuple<int, double, std::string> t = {1, 2.5, "hi"};
auto [a, b, c] = t;                 // a=1, b=2.5, c="hi"

struct Point { int x, y; };
Point p = {3, 4};
auto [x, y] = p;                     // structured binding

07 System & Build — files, preprocessor, correctness

File I/O fstream

Read/write files with ifstream (input), ofstream (output), and fstream (both). Supports text and binary modes.

std::ofstream out("file.txt");
out << "Hello file!\n";
out.close();

std::ifstream in("file.txt");
std::string line;
while (std::getline(in, line))
    cout << line;

// Binary mode
std::fstream f("data.bin",
    std::ios::binary | std::ios::in | std::ios::out);

Preprocessor #

Directives processed before compilation: #include (file insertion), #define (macros), include guards, and conditional compilation.

#include <iostream>           // standard header
#include "myheader.h"         // local header

#define PI 3.14159             // macro constant
#define MAX(a,b) ((a)>(b)?(a):(b)) // macro fn

#ifndef GUARD_H                // include guard
#define GUARD_H
// header content
#endif

#pragma once                  // modern guard

#ifdef DEBUG
cout << "debug mode";          // conditional
#endif

Const Correctness const

const protects variables, pointers (e.g., const int* vs int* const), member functions, and parameters from modification.

const int MAX = 100;           // cannot change
const int* p = &MAX;           // pointer to const int
int const c = 5;                 // same as const int

// const method
int getValue() const {         // promises not to modify *this
    return val;
}

// const & param — prevents copy + modification
void print(const std::string& s);

Union & Bit Fields union

union — all members share the same memory (size = largest member). Bit fields allow packing within a struct at the bit level.

union Value {
    int i;
    float f;
    char c[4];
};
Value v;
v.i = 42;                       // stores as int

struct Flags {
    unsigned int a : 1;        // 1 bit
    unsigned int b : 3;        // 3 bits
    unsigned int c : 4;        // 4 bits
};                                 // total: 8 bits
