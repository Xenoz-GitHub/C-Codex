Every C++ syntax construct, distilled into concise examples.

01 Fundamentals — core syntax & building blocks

Comments //

Single-line (//) and multi-line (/* */) comments. Entirely ignored by the compiler — used to document intent and disable code.

// This is a single-line comment
int x = 5;  // inline comment

/* Multi-line
   comment block */

/* Comments /* do NOT nest */  // error!

Variables & Data Types types

Fundamental types: int, double, char, bool, unsigned. Use auto for type deduction and const for immutables. size_t for sizes/indices.

int age = 25;
double pi = 3.14159;
char grade = 'A';
bool ready = true;
auto result = 42;          // deduced as int
const int MAX = 100;       // read-only constant
unsigned int n = 255u;
long long big = 1e15;
size_t sz = sizeof(arr);

Input / Output cin/cout

Console I/O via cout (output), cin (input), cerr (error), clog (log). getline reads whole lines.

cout << "Hello, " << name << "!\n";
cin >> name;                   // word input
getline(cin, line);             // line input
cerr << "Error occurred!";      // unbuffered error
clog << "Log: " << value;        // buffered log

Operators + - * /

Arithmetic, relational, logical, bitwise, ternary (?:), and compound assignment. Operator precedence matters — use parentheses to clarify.

int sum = a + b;         // arithmetic: + - * / %
bool eq = (a == b);      // relational: == != < > <= >=
bool ok = a && b || !c;  // logical: && || !
int bits = a & b | c;    // bitwise: & | ^ ~ << >>
int mx = (a > b) ? a : b; // ternary conditional

a += 5;  a -= 2;            // compound: += -= *= /= %=
++a;  b--;                   // increment / decrement

Enums & Structs enum/struct

enum — named integer constants. enum class (C++11) — scoped, type-safe. struct — grouped public data members.

enum Color { Red, Green, Blue };
enum class Status { Ok= 200, Err= 500 };
Status s = Status::Ok;          // scoped access

struct Point {
    double x, y;
};
Point p = {3.0, 4.0};

Type Casting cast

Four C++ casts: static_cast (safe), dynamic_cast (polymorphic), const_cast (remove const), reinterpret_cast (bit reinterpret). Avoid C-style (type)val.

double d = static_cast<double>(a) / b;

if (Derived* d = dynamic_cast<Derived*>(b))
    // null if cast fails

const_cast<char*>(str);      // strip const
reinterpret_cast<int*>(addr); // bit reinterpret

Namespaces namespace

Group identifiers to avoid collisions. Can be nested, aliased, or opened with using. The std namespace contains the standard library.

namespace Engine {
    namespace Math {
        int clamp(int v, int lo, int hi);
    }
}
using Engine::Math::clamp;       // single name
namespace EM = Engine::Math;     // alias
using namespace std;               // brings all (use sparingly!)

Type Aliases typedef/using

typedef creates type synonyms. using (C++11) is preferred — it supports alias templates and reads naturally left-to-right.

typedef unsigned long ulong;
using ulong = unsigned long;    // same, clearer

template<typename T>
using Vec = std::vector<T>;    // alias template
Vec<int> v;

Raw & Literal Suffixes R"()"

Raw string literals R"(...)" avoid escape sequences. C++11 user-defined literals (s, sv, i, f) add type suffixes.

// Raw string — no escapes needed
auto path = R"(C:\Program Files\)";

auto s = "hello"s;              // std::string
auto sv = "view"sv;             // std::string_view (C++17)
auto n = 42ull;                   // unsigned long long

02 Control Flow — branching & iteration

Conditionals if/else

if/else for boolean branching. C++17 adds if with initialiser — the variable stays in scope for the whole if/else chain.

if (x > 0) {
    // positive
} else if (x == 0) {
    // zero
} else {
    // negative
}

// C++17: init-statement
if (auto it = m.find(key); it != m.end()) {
    // use it — still in scope here
}

Switch switch/case

Multi-way branch on integral/enum types. Each case needs break or falls through. default catches unmatched values.

switch (grade) {
    case 'A': cout << "Excellent";
              break;
    case 'B':
    case 'C': cout << "Pass";      // fall-through
              break;
    default:  cout << "Fail";
}

// C++17: switch with init
switch (int cmd = getCmd(); cmd) {
    case 1: /* cmd still in scope */ break;
}

Loops for/while

Three classic loop forms: for (counter), while (pre-check), do-while (post-check, always runs once).

for (int i = 0; i < n; ++i) {
    // known iteration count
}
while (condition) {
    // checked before each iteration
}
do {
    // always runs at least once
} while (condition);

for (;;) { }                   // infinite loop

Range-based For : range

C++11 range-for iterates any sequence (arrays, containers, initializer lists). Use const auto& for read-only, auto& for mutation.

std::vector<int> v = {1,2,3};

for (int x : v) { }            // copy — expensive for large types
for (auto& x : v) { x *= 2; } // mutate in-place
for (const auto& x : v) { } // read-only, no copy

// C++17: structured bindings + range-for
for (const auto& [k, v] : my_map) { }

Recursion recurse

A function calling itself. Must have a base case to terminate. Each call adds a stack frame — deep recursion may overflow the stack.

int factorial(int n) {
    if (n <= 1) return 1;     // base case
    return n * factorial(n - 1); // recursive step
}

int fib(int n) {
    if (n <= 1) return n;
    return fib(n-1) + fib(n-2); // multiple calls
}

03 Data & Memory — storage, pointers, allocation

Arrays & Strings []

Fixed-size C-style arrays and the dynamic std::string. std::array (C++11) wraps fixed-size arrays with STL interface.

int arr[5] = {1,2,3};       // rest → 0
arr[0] = 10;
int len = sizeof(arr) / sizeof(arr[0]);

char msg[] = "hello";          // C-string (null-terminated)

std::string s = "C++";
s.size();  s.empty();  s += "17";
std::array<int, 3> a = {1,2,3};

Pointers & References * &

Pointers hold addresses; can be reassigned and be nullptr. References are aliases — must be initialised, cannot be rebound. Prefer references when possible.

int x = 42;
int* p = &x;           // pointer to x
cout << *p;            // dereference → 42

int& ref = x;          // reference (alias for x)
ref = 99;               // changes x to 99

p = nullptr;            // null pointer (C++11)
if (p != nullptr) { }  // or just if (p)

Dynamic Memory new/delete

Heap allocation with new/delete. Always match new with delete and new[] with delete[]. Prefer smart pointers.

int* p  = new int(42);    // single int on heap
delete p;

int* ar = new int[10];    // array on heap
delete[] ar;                // array delete — required!

// Smart pointers (C++11, <memory>)
auto u = std::make_unique<int>(5);
auto s = std::make_shared<int>(10);

04 Functions — reusable logic & callables

Functions ()

Declaration (prototype) + definition. Supports overloading (same name, different params), default arguments, pass-by-value/reference, and void returns.

int add(int a, int b) {     // definition
    return a + b;
}
void log(const std::string& msg,  // const ref = no copy
           int level = 0) { }       // default argument
double add(double a, double b);   // overload

Lambda Expressions []()

Inline anonymous functions (C++11). Capture [=] (by value), [&] (by reference), or selectively [x, &y]. mutable allows modifying captured values.

auto square = [](int x) { return x * x; };
cout << square(5);               // 25

int offset = 10;
auto add = [offset](int x) {       // capture by value
    return x + offset;
};
auto cnt = [n = 0]() mutable {      // init capture + mutable
    return n++;
};

// Generic lambda (C++14)
auto gen = [](auto a, auto b) { return a + b; };

Function Pointers fnptr

Store addresses of functions. The signature must match exactly. std::function (C++11) generalises to lambdas, bind, and any callable.

int (*op)(int, int) = &add;
op(3, 4);                         // call via pointer: 7

using BinOp = int(*)(int, int);

// std::function — any callable
std::function<int(int)> f = square;
f(3);  // works with lambdas, bind, function pointers

Inline & constexpr inline/constexpr

inline — hint to replace call with body (avoids overhead). constexpr — evaluated at compile-time. C++20 adds consteval (always compile-time).

inline int square(int x) {
    return x * x;
}

constexpr int fact(int n) {
    return (n <= 1) ? 1 : n * fact(n - 1);
}
constexpr int VAL = fact(5);  // compile-time

=default & =delete default/delete

= default — explicitly request compiler-generated version (ctor, dtor, copy/move). = delete — forbid a function (often copy ctor/assignment for unique resources).

class NoCopy {
public:
    NoCopy() = default;            // use compiler default
    ~NoCopy() = default;
    NoCopy(const NoCopy&) = delete;  // forbid copying
    NoCopy& operator=(const NoCopy&) = delete;
};

void onlyInt(int) { }              // OK
void onlyInt(double) = delete;     // forbid double calls

decltype & typeid decltype

decltype — yields the exact type of an expression at compile-time. typeid (from <typeinfo>) — runtime type identification (RTTI).

int x = 42;
decltype(x) y = 10;               // y is int

const int& cref = x;
decltype(cref) z = y;             // z is const int&

// decltype(auto) — perfect forwarding (C++14)
decltype(auto) forward(auto&& v) {
    return std::forward<decltype(v)>(v);
}

if (typeid(x) == typeid(int)) { }  // runtime type check

05 Object-Oriented Programming — classes, inheritance, polymorphism

Classes class

Encapsulation via private / public / protected. Constructors (with member initialiser lists), destructor, and the this pointer.

class Player {
private:
    int hp;                      // only accessible within class
public:
    Player(int h) : hp(h) { }     // member initialiser list
    ~Player() { }                  // destructor
    void takeDamage(int d) {
        hp -= d;
        this->hp = hp;            // explicit this->
    }
};  // semicolon required!

Inheritance :

Derive using :. Access modes: public (IS-A), protected, private. virtual inheritance prevents diamond duplicates. Base class destructor should be virtual.

class Entity {
public:
    int id;
    virtual ~Entity() = default;  // virtual dtor
};
class Player : public Entity { };
class Enemy  : public Entity { };

// Virtual inheritance
class D : public virtual B1, private B2 { };

Polymorphism virtual

Runtime dispatch via virtual functions. Pure virtual (= 0) makes a class abstract. override (C++11) ensures you actually override a base method.

class Shape {
public:
    virtual double area() const = 0; // pure virtual → abstract
    virtual ~Shape() = default;
};
class Circle : public Shape {
    double area() override {    // override checks base
        return 3.14159 * r * r;
    }
};
Shape* s = new Circle(5);
s->area();                // dynamic dispatch at runtime

Copy Control rule of 5

If a class manages a resource, define: destructor, copy ctor, copy assignment, move ctor, move assignment (Rule of Five, C++11). Missing one can cause resource leaks.

class Buffer {
    int* data;  size_t sz;
public:
    Buffer(size_t s) : sz(s), data(new int[s]) { }
    ~Buffer() { delete[] data; }          // 1. destructor
    Buffer(const Buffer& o)               // 2. copy ctor
        : sz(o.sz), data(new int[o.sz]) {
        std::copy(o.data, o.data+sz, data);
    }
    Buffer& operator=(const Buffer& o) {  // 3. copy assign
        if (this != &o) { /* copy + swap */ }
        return *this;
    }
    Buffer(Buffer&& o) noexcept         // 4. move ctor
        : sz(o.sz), data(o.data) { o.data = nullptr; }
    Buffer& operator=(Buffer&&) noexcept { } // 5. move assign
};

Operator Overloading operator

Define custom behaviour for operators on your types. Can be member or non-member functions. Common overloads: +, ==, <<, [], ().

struct Vec2 {
    float x, y;
    Vec2 operator+(const Vec2& o) const {
        return {x + o.x, y + o.y};
    }
    bool operator==(const Vec2& o) const {
        return x == o.x && y == o.y;
    }
};
Vec2 a = {1,2}, b = {3,4};
auto c = a + b;               // calls operator+

Static Members static

static members belong to the class, not instances. Must be defined outside the class. static local variables persist across function calls.

class App {
public:
    static int count;               // declaration
    static void showCount() {
        cout << count;              // static fn — no this
    }
};
int App::count = 0;                // definition (in .cpp)

int nextID() {
    static int id = 0;             // persists across calls
    return id++;
}

Friend friend

A friend function or class can access private/protected members of the class that declares it. Friendship is not inherited nor transitive.

class Account {
    int balance;
public:
    Account(int b) : balance(b) { }
    friend bool isOverdrawn(const Account& a);
    friend class Auditor;         // whole-class friendship
};
bool isOverdrawn(const Account& a) {
    return a.balance < 0;         // accesses private member
}

explicit & mutable explicit

explicit prevents implicit conversions via single-argument constructors or conversion operators. mutable allows modification in const methods (e.g., caches, mutexes).

class String {
public:
    explicit String(int n) { }     // no implicit int→String
};
String s = 5;                      // error: explicit
String s(5);                      // OK: direct init

struct Cache {
    mutable int hits = 0;
    int get() const {
        ++hits;                     // allowed: mutable
        return val;
    }
};

Conversion Operators operator T

Define implicit or explicit (explicit) conversions from your type to another type. Use explicit to avoid accidental conversions.

struct Rational {
    int num, den;
    operator double() const {      // implicit conversion
        return static_cast<double>(num) / den;
    }
    explicit operator bool() const {  // C++11: explicit
        return den != 0;
    }
};
Rational r{3, 4};
double d = r;                    // implicit: 0.75

06 Modern C++ — generics, safety, algorithms, utilities

Templates <T>

Generic code with type parameters. Supports function/class templates, specialisation, variadic templates (C++11), and template template parameters.

template<typename T>
T max(T a, T b) { return (a > b) ? a : b; }

template<typename T, int N>
class Array { T data[N]; };
template<> class Array<bool,8> { };  // specialisation

// Variadic (C++11)
template<typename... Args>
void print(Args... args) { (cout << ... << args); }

Exceptions try/catch

try/catch/throw for error handling. Catch by const reference. catch(...) catches anything. Standard exceptions inherit from std::exception.

try {
    if (error)
        throw std::runtime_error("operation failed");
} catch (const std::exception& e) {
    cout << "Error: " << e.what();
} catch (const char* msg) {
    // catch specific type
} catch (...) {
    // catch-all (any remaining exception)
}

STL Containers vector/map/set

Workhorses of the Standard Library: vector (dynamic array), map (key-value, sorted), set (sorted unique), unordered_map (hash-based, C++11).

std::vector<int> v = {1,2,3};
v.push_back(4);                  // O(1) amortised
v.pop_back();                    // remove last

std::map<std::string, int> m;
m["key"] = 42;                    // insert or update

std::set<int> s = {3,1,2};       // sorted, unique
std::stack<int> stk;              // LIFO adapter
std::queue<int> q;                 // FIFO adapter

Move Semantics &&

Rvalue references (&&) enable move semantics — transferring owned resources without copying. std::move casts lvalue → rvalue. std::forward preserves value category.

std::vector<int> a = {1,2,3};
std::vector<int> b = std::move(a);  // a is now empty

class MyType {
    int* ptr;
public:
    MyType(MyType&& other) noexcept  // move constructor
        : ptr(other.ptr) {
        other.ptr = nullptr;         // transfer ownership
    }
};

Iterators & Algorithms sort/find

Iterators abstract container access. <algorithm> provides sort, find, for_each, count_if, lower_bound, etc. Ranges (C++20) simplify further.

std::vector<int> v = {4,1,3,2};
std::sort(v.begin(), v.end());      // 1 2 3 4

auto it = std::find(v.begin(),
                    v.end(), 3);     // search

std::for_each(v.begin(), v.end(),
    [](int x) { cout << x; });      // lambda

int cnt = std::count_if(v.begin(),
    v.end(), [](int x) { return x > 2; });

optional & variant C++17

std::optional — may or may not hold a value (safe nullable). std::variant — type-safe union of known types (never empty). Both are value types (no heap allocation).

std::optional<int> parseInt(const std::string& s) {
    if (s.empty()) return std::nullopt;
    return std::stoi(s);
}
if (auto val = parseInt("42"))
    cout << *val;                   // 42

std::variant<int, float, std::string> v;
v = "hello";                        // holds string
cout << std::get<std::string>(v);  // "hello"

Structured Bindings C++17

Decompose tuples, pairs, arrays, and structs into named variables in one line (C++17). Works with auto, references, and const.

// With map iteration
std::map<std::string, int> scores;
for (const auto& [name, score] : scores) {
    cout << name << ": " << score;
}

// With tuple
auto [a, b, c] = std::tuple{1, 2.5, "hi"};

// With struct
struct Point { int x, y; };
Point p{3, 4};
auto [x, y] = p;

pair & tuple pair/tuple

std::pair — holds two values (e.g., map elements). std::tuple (C++11) — heterogeneous fixed-size collection. Access with std::get or structured bindings.

std::pair<int, std::string> p = {1, "one"};
cout << p.first << p.second;      // .first / .second

std::tuple<int, double, char> t{1, 2.5, 'x'};
cout << std::get<0>(t);             // get by index
cout << std::get<double>(t);       // get by type

auto [a, b, c] = t;                 // structured binding (C++17)

Time & Chrono chrono

<chrono> (C++11) provides type-safe time utilities: duration, time_point, high_resolution_clock. C++20 adds calendar dates and time zones.

using namespace std::chrono;

auto start = high_resolution_clock::now();
// ... some work ...
auto end = high_resolution_clock::now();

auto ms = duration_cast<milliseconds>(end - start);
cout << ms.count() << "ms\n";

// Sleep
this_thread::sleep_for(seconds(1));

string_view & span C++17/20

std::string_view (C++17) — non-owning view of a character sequence (no allocation). std::span (C++20) — non-owning view of a contiguous sequence.

std::string_view trim(std::string_view s) {
    while (s.front() == ' ') s.remove_prefix(1);
    while (s.back()  == ' ') s.remove_suffix(1);
    return s;                         // no allocation!
}
cout << trim("  hello  ");            // "hello"

// span (C++20)
void process(std::span<int> data) {
    for (int& x : data) x *= 2;
}

if constexpr C++17

Compile-time conditional branching. The discarded branch is not instantiated — useful for templates to avoid compilation errors for unused code.

template<typename T>
auto stringify(const T& v) {
    if constexpr (std::is_arithmetic_v<T>) {
        return std::to_string(v);      // only for numbers
    } else {
        return std::string{v};           // only for strings
    }
}

// Unlike runtime if, discard is NOT compiled
cout << stringify(42);                // "42"
cout << stringify("hello");          // "hello"

Fold Expressions C++17

Apply a binary operator over a parameter pack. Four forms: unary left/right, binary left/right. Essential for variadic template utilities.

template<typename... Args>
auto sum(Args... args) {
    return (args + ...);               // unary right fold
}
// sum(1, 2, 3) → (1 + (2 + 3)) → 6

template<typename... Args>
void printAll(Args... args) {
    (cout << ... << args) << '\n';     // binary left fold
}

template<typename... Args>
bool allTrue(Args... args) {
    return (... && args);              // unary left fold
}

07 System & Build — files, preprocessor, correctness

File I/O fstream

File streams: ifstream (read), ofstream (write), fstream (both). Supports text and binary modes. Always check if the file opened successfully.

std::ofstream out("file.txt");
if (out) {                         // check opened
    out << "Hello file!\n";
    out.close();
}

std::ifstream in("file.txt");
std::string line;
while (std::getline(in, line))     // read line by line
    cout << line;

// Binary mode
std::fstream f("data.bin",
    std::ios::binary | std::ios::in | std::ios::out);

Preprocessor #

Directives processed before compilation: includes, macros, include guards, #pragma once, #ifdef for platform detection, # and ## operators.

#include <iostream>           // standard header
#include "myheader.h"         // local header

#define PI 3.14159             // macro constant
#define MAX(a,b) ((a)>(b)?(a):(b)) // macro fn (parentheses!)

#ifndef GUARD_H                // traditional include guard
#define GUARD_H
#endif

#pragma once                  // modern guard (no name needed)
#ifdef _WIN32                  // platform detection
    // Windows code
#endif

Const Correctness const

const prevents modification. Read pointer declarations right-to-left: const int* p = pointer to const int; int* const p = const pointer to int.

const int MAX = 100;           // cannot modify
const int* p = &MAX;           // pointer to const int
int const c = 5;                 // same as const int

// const methods — promise not to modify *this
int getValue() const { return val; }

// const& param — avoids copy, prevents modification
void print(const std::string& s);

Union & Bit Fields union

union — all members share the same memory address (size = largest member). Bit fields specify exact bit widths within a struct/union.

union Value {
    int i;
    float f;
    char c[4];
};                                  // sizeof(Value) == 4
Value v;
v.i = 42;                         // stores as int

struct Flags {
    unsigned int a : 1;          // 1-bit field
    unsigned int b : 3;          // 3-bit field
    unsigned int c : 4;          // 4-bit field
};                                   // packed into 8 bits total

08 Advanced C++ — Lvalues, concepts, placement

Lvalues & Rvalues & &&

Lvalues — have an address (variables, references). Rvalues — temporary values (literals, std::move). const& binds both; && binds only rvalues.

int x = 10;                     // x is lvalue, 10 is rvalue
int& lref = x;                  // lvalue reference
int&& rref = 20;                // rvalue reference (C++11)

// Forwarding reference (T&&) — preserves category
template<typename T>
void wrapper(T&& v) {
    other(std::forward<T>(v));    // perfect forwarding
}

// std::move converts lvalue to rvalue
std::vector<int> b = std::move(a);

Concepts (C++20) requires

Named compile-time constraints on template parameters. Clearer errors than SFINAE. Predefined: std::integral, std::floating_point, std::ranges::range, etc.

template<typename T>
concept Addable = requires(T a, T b) {
    { a + b } -> std::convertible_to<T>;
};

template<Addable T>                // constrained template
T add(T a, T b) { return a + b; }

// Using predefined concepts
auto square(std::integral auto x) {  // abbreviated syntax
    return x * x;
}

Placement new & Allocators placement

Construct an object at a pre-allocated memory address. No allocation — just construction. Must call destructor manually. Used in custom allocators, std::vector internals.

// Allocate raw memory
void* mem = operator new(sizeof(std::string));

// Construct at that address
std::string* s = new (mem) std::string("hello");

// Use it
cout << *s;

// Destroy manually
s->~basic_string();
operator delete(mem);           // free raw memory

// Nothrow new — returns nullptr on failure
int* p = new (std::nothrow) int(42);
if (!p) { /* handle */ }
